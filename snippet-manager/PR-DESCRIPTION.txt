## 🎯 主要改进

### 1. 💾 数据库持久化到宿主机

**改进内容：**
- 将数据库从 Docker volume 改为 bind mount
- 数据库文件映射到 `./data/database/snippets.db`
- 支持所有环境配置（dev/prod/default）

**配置变更：**
```yaml
# 旧配置（Docker volume）
volumes:
  - backend-data:/app/instance

# 新配置（Bind mount）
volumes:
  - ./data/database:/app/instance
```

**优势：**
- ✅ 升级安全：容器更新不影响数据
- ✅ 数据可见：可直接访问和备份数据库文件
- ✅ 易于迁移：复制 data/ 目录即可迁移数据

**涉及文件：**
- `docker-compose.yml` - 生产环境配置
- `docker-compose.prod.yml` - 预构建镜像配置
- `docker-compose.dev.yml` - 开发环境配置
- `.gitignore` - 排除 data/ 目录

## 🚀 使用方法

### 运行测试
```bash
cd snippet-manager

# 运行所有测试
./run-tests.sh

# 只运行单元测试
./run-tests.sh unit

# 只运行集成测试（需要服务运行）
./run-tests.sh integration

# 测试数据库挂载
./run-tests.sh database
```

### 查看数据库
```bash
# 数据库位置
ls -lh ./data/database/snippets.db

# 备份数据库
cp -r data/database backups/database-$(date +%Y%m%d)
```

### 启动服务
```bash
# 数据库会自动创建在 ./data/database/
./docker-start.sh
```
